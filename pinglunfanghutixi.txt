# 宝妈说 - 评论区防护体系

## 核心原则：评论区是内容的延伸，质量标准不能降低

---

## 一、评论发布时的实时拦截

### 1. 软文评论识别库

**高频骚扰词汇**（需要持续维护）
```
一级违规词（直接拦截）：
- "微信xxxxx""加我xxxxx""私聊我"
- "有需要可以联系我"
- "我也用过这款，效果真的绝了"（刻板推荐）
- "同款链接""购买链接""官方店铺"
- "代购""微商""一手货源"
- "@某品牌 官方旗舰店"

二级违规词（标记审核）：
- "强烈推荐""必买""回购""絕了"
- "医生推荐""权威认证"
- "仅限今天""限时优惠"
- 具体数字+人民币符号的组合（"89元""¥199"）
- 品牌名+具体型号+优惠词（"xxx奶粉 新客88折"）
```

**识别规则**
- 触发一级词 → 直接拦截，不发布，用户收到提示："你的评论含有不适合的内容"
- 触发二级词 → 标记为"待审核"，用户看不到，等人工审核
- 规则不是绝对的，比如"推荐"这个词本身没问题，但"强烈推荐+品牌名"的组合就是问题

---

### 2. 账号风险评分（评论端）

**账号信息识别**
```
高风险特征：
- 账号年龄<3天，发评论
- 账号无头像、无个人信息、无宝宝信息
- 账号名字是"微商""代理""商城"等营销词汇
- 账号从未发布过任何笔记，只评论
- 账号在24小时内评论>50条
- 评论全部指向同一个商品/品牌
```

**风险等级判定**
- 低风险账号：评论直接发布
- 中风险账号：评论需要人工审核（2小时内）
- 高风险账号：评论直接隐藏，发布者看不到

---

### 3. 评论内容的多维分析

**文本分析**
- 长度检测：评论<10字 → 可能是刷屏/水军，标记
- 相似度检测：同一账号的多条评论高度重复 → 标记为"可能是机器发送"
- 链接检测：评论中包含链接 → 直接拦截（除非是平台内链接）
- 特殊符号：评论含大量emoji、特殊符号、重复字符 → 标记（垃圾信息特征）

**示例识别**
```
❌ 被拦截：
"我家宝宝用了这款真的绝了，微信18xxxxx有需要联系我"

❌ 被标记审核：
"我也推荐这款产品，效果很好，现在还有活动👍"

✅ 通过：
"我家宝宝也用过这个，但坚持了一个月才见效，因人而异吧"

✅ 通过：
"请问这款产品在哪里买的？我想试试"
```

---

### 4. 评论发布前的选择确认

**强制提醒机制**
当检测到评论可能是"推荐"性质时，系统弹窗：

```
⚠️ 提示：你的评论可能包含产品推荐

是否为真实使用体验分享？
□ 是，我真的用过并想分享经验
□ 否，让我重新编辑

如果你经常发送推荐性评论，账号可能被限制发布权限。
```

用户必须确认，否则评论不发布。这个"确认"会被记录。

---

## 二、发布后的异常检测

### 1. 评论互动异常监测

**点赞异常**
- 某条评论在短时间内（比如1小时）获得异常多的点赞
  - 正常评论：点赞增长缓慢、线性
  - 异常评论：点赞突增（可能是买赞）
- 检测规则：某条评论的点赞数在1小时内>50 → 自动标记审核

**回复异常**
- 如果某条评论引发大量重复回复（比如10个几乎一样的回复）
  - 迹象：可能这条评论本身是不当内容，吸引了水军
  - 处理：把这整条"话题链"标记为低优先级，不在前排显示

**评论者异常**
- 同一个账号在24小时内：
  - 在同一条笔记下发5条以上评论 → 标记
  - 在不同笔记下发同样内容的评论>3次 → 标记为"机器刷屏"
  - 在同一笔记的评论区中，与其他2个账号互相点赞/回复 → 可能是"评论小分队"

---

### 2. 评论链路的监测

**链式推广识别**
```
风险模式 1："钓鱼链"
原文："我家宝宝用了这个发育指标很好"
↓
评论1："请问在哪里买的？效果怎样？"（看似真实提问）
↓
评论2："我有微商代理，微信xxxxx"（暴露）
↓
系统判定：这是有组织的"钓鱼推广链"
处理：把这整条链路的评论都隐藏，账号标记高风险

风险模式 2："话题导向"
原文："分享我家宝宝用过的产品"
↓
评论1："强烈推荐xx品牌"
评论2："xx品牌真的好用"
评论3："我也是xx的忠实用户"
↓
系统判定：多个账号在同一条笔记下重复推荐同一品牌
处理：这条笔记的评论区自动降权，这些账号标记为"营销号"
```

---

## 三、评论的展示策略

### 1. 评论排序不是"最新"和"热度"

**宝妈说的评论排序优先级**（重要！这是差异化）

```
第一优先级：笔记作者的回复
- 原作者的任何回复都排在最前
- 为什么？鼓励互动，让提问者能快速得到答案

第二优先级：有价值的提问
- "请问在哪里买的？"
- "宝宝多大时可以用？"
- "有没有副作用？"
- 系统自动识别常见的"有价值提问"

第三优先级：作者点赞/回复过的评论
- 说明作者认可这个评论
- 增加互动，减少刷屏

第四优先级：认证宝妈/医生的评论
- 专业人士的意见排靠前
- 但前提是这条评论没有推广嫌疑

第五优先级：其他高赞评论
- 但要过滤掉买赞的
- 使用"赞增长曲线"判断：自然增长 vs 突增

最后：低质/可疑评论
- 包括：重复内容、可能推广、低价值、侮辱性评论
```

**展示限制**
- 默认只显示前10条评论 + 3条"热门回复"
- 用户可以点"加载更多"
- 可疑评论（待审核的）在"加载更多"之后才会显示
- 被用户举报超过3次的评论，在排序中自动降到最后

---

### 2. 评论的可见性控制

**分级可见**
```
绿灯评论（立即显示）：
- 普通有价值问答
- 真实经验分享
- 原作者的回复

黄灯评论（标记但显示）：
- 包含可能推荐词汇但通过审核的
- 显示时自动添加标签："用户分享"（提醒其他人这是推荐内容）
- 示例："用户分享：我也推荐这款产品" [标签]

红灯评论（隐藏）：
- 待审核的
- 被举报多次的
- 低质量内容
- 仅作者和审核员可见
```

## 四、用户举报机制（评论专项）

### 1. 便捷的举报入口

**每条评论的右侧都有三个按钮**
```
👍 赞
💬 回复
⋯ 更多
  └─ 举报 ← 用户点这里
```

**举报分类（评论专项）**
- 广告/推广（最常见）
- 推荐不安全产品
- 骚扰/骂人
- 涉及隐私
- 重复刷屏
- 其他

**举报必填项**
- 具体指出评论中的哪句话是问题
- 用户可以标注评论中的部分文本
- 这样审核员能快速定位问题

---

### 2. 举报反馈与激励

**实时反馈**
- 举报后，用户看到："已提交，我们会在1小时内审核"
- 如果被处理，用户收到通知："感谢举报，该评论已删除"

**举报激励**
- 举报得当的用户（举报正确率>80%）获得"社区守护者"徽章
- 每月有效举报>5次，可获得平台"月度守护者"认可
- 展示在用户的个人页面上（增加荣誉感）

**举报滥用处罚**
- 举报成功率<20%（乱举报）的账号，举报权限被限制
- 连续恶意举报（明显正常内容被标记为举报）→ 警告 → 禁止举报权限

## 五、评论区内容的长期运营

### 1. 定期评论审计

**每周审计计划**
- 随机抽取当周所有笔记的评论区
- 抽样率：初期50%，后期20%
- 检查项目：
  - 有没有漏掉的广告评论
  - 评论排序是否合理
  - 有没有"评论小分队"的迹象

**月度评论报告**
- 统计数据：
  - 总评论数、有效评论率
  - 被删除评论数及原因分布
  - 被举报最多的几类问题
  - 高风险账号数量
- 对外透明发布（建立信任）

---

### 2. 评论社区氛围建设

**鼓励正向评论**
- 原作者点赞的评论显示"作者认可" [徽章]
- 有价值的提问被高赞时，显示"高价值讨论" [徽章]
- 这样激励用户发更多有意义的评论

**打压负向评论**
- 频繁发广告的账号，其所有评论自动标记为"待审核"
- 评论被删除次数>5次的账号，新评论需要先审核再发布

**创建评论规范指南**
- 发布"宝妈说评论区礼仪"
- 举例：
  - ✅ 好评论："我也用过，但建议6个月后再用"
  - ❌ 坏评论："我家的同款，微信xxxxx"
- 让用户理解什么是优质评论

## 六、技术实现方案

### 1. 评论审核的AI模型

**多模型组合**
```
模型1：文本分类（NLP）
- 输入：评论文本
- 输出：分类标签（推荐、提问、吐槽、骂人等）
- 准确率目标：>85%

模型2：关键词匹配（规则引擎）
- 输入：评论文本
- 输出：是否包含风险词汇
- 处理：快速过滤明显广告

模型3：账号风险评分（行为分析）
- 输入：发布者的账号历史
- 输出：风险分数（1-100）
- 考虑因素：账号年龄、发布频率、被删除率等

模型4：链路识别（图算法）
- 输入：评论间的回复关系、点赞关系
- 输出：识别"评论小分队""钓鱼链"
- 处理：自动标记可疑链路
```

**初期实现**
- 不需要全部自研，可用现成的API：
  - 阿里云内容安全API（¥0.001/次，检测敏感词）
  - 腾讯云文本内容审核（¥0.002/次）
  - 百度内容审核服务（¥0.001/次）
- 再叠加自己的关键词库（母婴营销词汇）

**成本估算**
- 100万条评论/月 × ¥0.001-0.002 = ¥1000-2000/月
- 初期成本很低

---

### 2. 评论区的数据监控

**需要实时追踪的指标**
```
评论质量指标：
- 日均评论数、增长趋势
- 有效评论率（非垃圾信息）
- 评论被删除率、原因分布
- 平均评论长度（过短=水军）

异常检测指标：
- 高赞评论的赞增长曲线
- 同账号重复评论频率
- 评论中的URL链接数量
- 新账号评论占比

用户参与度指标：
- 评论回复率（有人回复的评论占比）
- 原作者参与度（回复了多少评论）
- 每条笔记的平均讨论深度
```

**告警规则**
- 某条评论的赞在1小时内>50 → 自动审核
- 某账号24小时评论>100条 → 冻结账号
- 某笔记的评论区被举报>3次 → 自动降权

## 八、评论区常见问题处理

### Q1：会不会过度审核，打击用户积极性？

**答**：评论的拦截主要集中在"明显广告"，比如含有微信号、代购链接。正常的推荐意见（比如"我也推荐这款产品"）会标记但不拦截。需要在运营中不断调整敏感度。

建议：
- 初期设置敏感度较低（宽松）
- 每周查看被拦截评论日志，看有没有误杀
- 逐步调整

### Q2：怎么处理"隐晦推广"（不含敏感词但明显是广告）？

**答**：这是最难的部分，需要人工+算法结合。

例子：
- 评论："我也给宝宝用这个，效果超好，宝宝睡眠改善了"
- 看似正常，但如果发现这个账号在10个不同的笔记下都说同样的话，就是问题

处理：
- 依赖于"账号历史行为分析"
- 如果某账号的评论高度重复 → 自动标记为可疑
- 需要人工审核员定期抽查

### Q3：评论被删除后，用户会不会投诉？

**答**：会的。所以需要：
1. 透明的申诉机制（用户可以申诉被删除的评论）
2. 清晰的社区规范（用户知道为什么被删）
3. 申诉响应快（1小时内回复）
4. 申诉成功率不要太低（应该>15%，说明系统不是绝对的）

## 九、与笔记内容防控的联动

**重要：评论审核不能孤立进行**

如果某条笔记本身就是软文（已标记），那么：
- 允许在下面有推荐性评论（因为笔记本身已经被标记了）
- 但要标记清楚："下列评论为用户推荐分享"
- 这样透明度高，用户自己判断

如果某条笔记是正常内容，但评论区全是推广：
- 笔记本身不动
- 评论区的推广内容全部删除
- 笔记的信誉值不变

## 十一、防控核心要点（老生常谈但重要）

1. **从源头防**：评论发布时就过滤，不要事后补救
2. **算法+人工**：不能只靠AI，人工审核很必要
3. **链路识别**：不只看单条评论，要看账号和评论间的关系
4. **透明度**：告诉用户你在怎么治理，建立信任
5. **激励机制**：让好的评论被推荐，让坏的评论被埋没
6. **不断迭代**：营销手段在变化，你的防守也要变化
