### 模块4：AI症状自查

#### 4.1 症状描述
- 快捷症状选项
- 文字详细描述
- 语音输入
- 上传照片（最多6张）

#### 4.2 AI分析
- AI文本分析
- AI图像识别（分开处理，降低延迟）
- 生成分析报告（可能原因+建议）
- 严重程度评估
- 就医建议
- 历史症状记录
- 症状趋势分析

**实现策略**：

**Phase 0（0-3个月）**：极简实现
- 文本分析：基于规则匹配（呕吐→毛球症/消化不良）
- 图像识别：仅模板匹配（对比预设的症状图片库）
- 开发成本极低（2-3天）
- 响应快速（<0.5秒）
- 准确率：60-70%

**Phase 1（3-6个月）**：引入深度学习
- 图像识别：深度学习模型
- 文本分析：AI模型
- 准确率提升至80%+

**Phase 2（6-12个月）**：全面AI化
- 多模态融合（文本+图像）
- 准确率提升至85%+
- 支持更多症状类型

#### 4.3 商品推荐

**CPS转化闭环**：

**来源标签**：
- `ai_health_check`：AI症状自查
- `calendar_reminder`：健康日历提醒
- `content_share`：内容种草
- `feed_recommendation`：Feed流推荐
- `search`：搜索

**数据追踪**：
```
每个商品点击都记录来源标签
↓
统计各场景的转化率和GMV
↓
分析热销品类（Top 3品类占比）
↓
为Phase 2自有品牌选品提供数据支撑
```

**数据分析示例**：
```
AI症状自查 → 药品类 → 转化率8% → GMV占比40%
健康日历 → 疫苗/驱虫 → 转化率10% → GMV占比35%
内容种草 → 主粮/零食 → 转化率5% → GMV占比25%

结论：优先做药品、疫苗/驱虫、主粮类自有品牌
```

- 配方推荐（根据品种、年龄、体重）
- 用量计算
- 动态调整（根据体重变化）
- 组合订阅（猫粮+猫砂+零食套餐）

**订阅周期**：
- 猫粮/狗粮：15天/30天
- 猫砂：15天/30天
- 驱虫药：30天/90天

**智能化分阶段实现**：

**Phase 2（6-12个月）**：规则匹配 + 简单推荐
```
推荐逻辑：
1. 品种匹配：金毛 → 大型犬粮
2. 年龄匹配：3个月 → 幼犬粮，8岁 → 老年粮
3. 用量计算：体重 × 3% = 日用量
4. 周期推荐：日用量 × 30 > 5kg → 30天周期
```

**Phase 3（12个月+）**：动态调整
```
引入体重曲线数据：
- 体重持续增加 → 减少10%用量
- 体重持续减少 → 增加10%用量
- 体重稳定 → 保持当前用量

引入健康数据：
- 有慢性病 → 推荐处方粮
- 有过敏史 → 排除过敏成分
```

**好处**：
- Phase 2 快速上线（开发5天）
- Phase 3 精准推荐（提升复购率）

## ⚠️ 风险控制策略

### 技术风险降级方案

**AI功能降级**：
- AI识别失败 → 手动选择
- AI分析失败 → 规则匹配
- AI推荐失败 → 简单排序

### 商业风险控制

**自有品牌失败应对**：
- 小批量测试（100-500单）
- 快速迭代（2周一版）
- 及时止损（测试失败立即停止）

**止损点**：
- 测试转化率 < 1% → 停止
- 用户反馈差评 > 30% → 停止
- 复购率 < 10% → 停止

### 运营风险控制

**线下服务质量**：
- 严格审核商家资质
- 用户评价体系
- 差评快速响应（2小时内）
- 定期服务质量检查
